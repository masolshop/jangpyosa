# 🎯 기업관리자 알림 표시 문제 - 최종 해결

## 📊 문제 상황

**사용자 보고**: "기업관리자에 알림이 안떠"

### 증상
1. 기업관리자 화면에서 알림 아이콘에 개수가 표시되지 않음
2. 업무지시 상세보기에서 확인 리스트가 0으로 표시
3. 공지사항, 근태 알림도 표시되지 않음

---

## 🔍 원인 분석

### ✅ 백엔드는 정상
```bash
# 실제 데이터 확인
회사: 페마연구소
  → 직원: 16명
  → 업무지시: 14개  
  → 공지사항: 18개
  → 근태기록: 18건
  → 알림: 30개 (미읽음 25개)

# API 응답도 정상
GET /notifications?limit=5
{
  "notifications": [...],
  "total": 30,
  "unreadCount": 25
}

GET /notifications/unread-count?byType=true
{
  "total": 25,
  "byType": {
    "WORK_ORDER_COMPLETED": 1,
    "ATTENDANCE_ISSUE": 24,
    ...
  }
}
```

### ❌ 프론트엔드 문제 발견

**1. API 호출 시 필수 파라미터 누락**
```typescript
// ❌ 이전 코드 (apps/web/src/components/Sidebar.tsx:40)
const url = `${API_BASE}/notifications/unread-count`;
//                                                    ← byType=true 없음!

// ✅ 수정 후
const url = `${API_BASE}/notifications/unread-count?byType=true`;
```

**2. 신규 알림 타입 미포함**
```typescript
// ❌ 이전 코드 - WORK_ORDER만 카운트
workOrder: byType.WORK_ORDER || 0

// ✅ 수정 후 - WORK_ORDER_COMPLETED 포함
workOrder: (byType.WORK_ORDER || 0) + (byType.WORK_ORDER_COMPLETED || 0)

// ❌ 이전 코드 - ANNOUNCEMENT만 카운트
announcement: byType.ANNOUNCEMENT || 0

// ✅ 수정 후 - ANNOUNCEMENT_READ 포함
announcement: (byType.ANNOUNCEMENT || 0) + (byType.ANNOUNCEMENT_READ || 0)
```

**3. 읽음 처리 시 신규 타입 누락**
```typescript
// ❌ 이전 코드
onNotificationClear={() => markNotificationsByTypeAsRead(['WORK_ORDER'])}

// ✅ 수정 후
onNotificationClear={() => markNotificationsByTypeAsRead(['WORK_ORDER', 'WORK_ORDER_COMPLETED'])}
```

## 🛠️ 해결 방법

### 수정 파일: `apps/web/src/components/Sidebar.tsx`

#### 1. API URL에 `byType=true` 추가
```diff
- const url = `${API_BASE}/notifications/unread-count`;
+ const url = `${API_BASE}/notifications/unread-count?byType=true`;
```

#### 2. 알림 카운트 로직 업데이트
```diff
  const counts = {
    total: data.total || 0,
    leave: (byType.LEAVE_REQUEST || 0) + (byType.LEAVE_APPROVED || 0) + (byType.LEAVE_REJECTED || 0),
-   workOrder: byType.WORK_ORDER || 0,
+   workOrder: (byType.WORK_ORDER || 0) + (byType.WORK_ORDER_COMPLETED || 0),
-   announcement: byType.ANNOUNCEMENT || 0,
+   announcement: (byType.ANNOUNCEMENT || 0) + (byType.ANNOUNCEMENT_READ || 0),
    attendance: (byType.ATTENDANCE_REMINDER || 0) + (byType.ATTENDANCE_ISSUE || 0),
  };
```

#### 3. 읽음 처리 타입 업데이트
```diff
- onNotificationClear={() => markNotificationsByTypeAsRead(['WORK_ORDER'])}
+ onNotificationClear={() => markNotificationsByTypeAsRead(['WORK_ORDER', 'WORK_ORDER_COMPLETED'])}

- onNotificationClear={() => markNotificationsByTypeAsRead(['ANNOUNCEMENT'])}
+ onNotificationClear={() => markNotificationsByTypeAsRead(['ANNOUNCEMENT', 'ANNOUNCEMENT_READ'])}
```

- **실시간 토스트**: 새 알림 발생 시 토스트 알림

## 📊 알림 타입별 구조

### 기업관리자가 받는 알림

| 타입 | 트리거 | 표시 위치 | 카운트 |
|------|--------|----------|--------|
| `LEAVE_REQUEST` | 직원 휴가 신청 | 휴가관리 | ✅ |
| `LEAVE_APPROVED` | 휴가 승인 | 휴가관리 | ✅ |
| `LEAVE_REJECTED` | 휴가 거절 | 휴가관리 | ✅ |
| `WORK_ORDER` | 업무지시 생성 | 업무관리 | ✅ |
| `WORK_ORDER_COMPLETED` | 업무 50% 완료 | 업무관리 | ✅ (신규) |
| `ANNOUNCEMENT` | 공지사항 생성 | 공지관리 | ✅ |
| `ANNOUNCEMENT_READ` | 공지 50% 읽음 | 공지관리 | ✅ (신규) |
| `ATTENDANCE_ISSUE` | 지각/조퇴/결근 | 근태관리 | ✅ |
| `ATTENDANCE_REMINDER` | 출근 알림 | 근태관리 | ✅ |

### 직원이 받는 알림

| 타입 | 트리거 | 표시 위치 |
|------|--------|----------|
| `LEAVE_APPROVED` | 휴가 승인 | 직원 대시보드 |
| `LEAVE_REJECTED` | 휴가 거절 | 직원 대시보드 |
| `WORK_ORDER` | 새 업무지시 | 직원 업무 |
| `ANNOUNCEMENT` | 새 공지사항 | 직원 공지 |
