## 🔗 ID 구조 설명

### 3단계 ID 연결 구조

```
Company (회사)
    ↓ companyId
BuyerProfile (구매자 프로필)
    ↓ buyerId
DisabledEmployee (장애인 직원)
    ↓ employeeId
User (로그인 계정, role=EMPLOYEE)
```

### 각 테이블의 역할

1. **Company**: 회사 기본 정보 (이름, 사업자번호, 타입)
2. **BuyerProfile**: 고용 의무 관련 정보 (총 직원 수, 장애인 직원 수)
3. **DisabledEmployee**: 장애인 직원 상세 정보 (중증도, 근로시간, 급여 등)
4. **User**: 로그인 계정 (전화번호, 비밀번호, 권한)

### employeeId 연동의 의미

- `User.employeeId` = `DisabledEmployee.id`로 연결되어야 함
- 연동 완료 시: 장애인 직원이 자신의 업무지시, 급여, 근태 등을 조회 가능
- 미연동 시: 로그인 계정이 없어서 시스템 사용 불가
