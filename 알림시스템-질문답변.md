# 알림 시스템 질문 답변

## 질문: "기업관리자 알림에서 장애인직원근태관리, 장애인직원업무관리 알림 로직이 어떻게 되지?"

### 답변:

## 1. 장애인직원근태관리 (Attendance) 알림

### 현재 상태: ⚠️ **구현 예약됨** (아직 미구현)

근태 관리 알림은 다음과 같이 설계되었으나 아직 구현되지 않았습니다:

```typescript
// 예정된 로직 (apps/api/src/routes/attendance.ts에 추가 필요)

// 1. 출근 시간 체크 (09:00 기준)
router.post('/check-late-arrivals', requireAuth, async (req, res) => {
  const now = new Date();
  const hour = now.getHours();
  
  // 09:30 이후에 출근 기록이 없는 직원 찾기
  if (hour >= 9 && minutes >= 30) {
    const lateEmployees = await prisma.disabledEmployee.findMany({
      where: {
        buyerId: req.user!.companyId,
        // 오늘 출근 기록이 없는 직원
        NOT: {
          attendanceRecords: {
            some: {
              date: today,
              clockIn: { not: null }
            }
          }
        }
      }
    });
    
    // 관리자에게 알림
    if (lateEmployees.length > 0) {
      await notifyAttendanceIssue(
        buyerId,
        `${lateEmployees.length}명의 직원이 출근하지 않았습니다`,
        'late'
      );
    }
  }
});

// 2. 조퇴 체크
router.post('/clock-out', requireAuth, async (req, res) => {
  const now = new Date();
  const hour = now.getHours();
  
  // 17:00 이전에 퇴근하면 조퇴
  if (hour < 17) {
    // 관리자에게 알림
    await notifyAttendanceIssue(
      buyerId,
      `${employee.name}님이 조퇴했습니다 (${now.toTimeString()})`,
      'early_leave'
    );
  }
});
```

### 구현 필요 사항:
1. **지각 알림**: 09:30 이후 출근 기록 없는 직원 → 관리자 알림
2. **조퇴 알림**: 17:00 이전 퇴근 기록 → 관리자 알림
3. **무단결근 알림**: 하루 종일 출근 기록 없음 → 관리자 알림

## 2. 장애인직원업무관리 (Work Order) 알림

### 현재 상태: ✅ **완전 구현됨**

업무 관리 알림은 두 가지 방향으로 작동합니다:

### A. 관리자 → 직원 알림

**트리거**: 관리자가 새 업무지시를 생성할 때

```typescript
// apps/api/src/routes/work-orders.ts
router.post('/', requireAuth, async (req, res) => {
  // 1. 업무지시 생성
  const workOrder = await prisma.workOrder.create({
    data: {
      title,
      content,
      companyId,
      buyerId,
      // ...
    }
  });
  
  // 2. 모든 직원에게 알림 전송
  await notifyNewWorkOrder(
    workOrder.buyerId,
    workOrder.id,
    workOrder.title
  );
  // ↓ 내부적으로:
  // - 회사의 모든 DisabledEmployee를 조회
  // - 각 직원의 userId로 WORK_ORDER 알림 생성
  // - 알림 내용: "📋 새로운 업무: {title}"
});
```

**결과**: 직원 앱에서 즉시 WORK_ORDER 알림을 받습니다.

---

### B. 직원 → 관리자 알림 (50% 완료 시)

**트리거**: 직원이 업무 완료 처리를 할 때마다 체크

```typescript
// apps/api/src/routes/work-orders.ts
router.post('/:id/confirm', requireAuth, async (req, res) => {
  // 1. 직원이 업무 완료 처리
  const confirmation = await prisma.workOrderConfirmation.create({
    data: {
      workOrderId: id,
      companyId,
      buyerId,
      employeeId,
      userId,
      note: req.body.note
    }
  });
  
  // 2. 완료율 계산
  const totalEmployees = await prisma.disabledEmployee.count({
    where: { buyerId: workOrder.buyerId }
  });
  
  const confirmCount = await prisma.workOrderConfirmation.count({
    where: { workOrderId: id }
  });
  
  const confirmPercentage = (confirmCount / totalEmployees) * 100;
  
  // 3. 50% 이상 완료 시 관리자에게 알림 (1회만)
  if (confirmPercentage >= 50 && !alreadyNotified) {
    await notifyWorkOrderCompleted(
      workOrder.buyerId,
      workOrder.id,
      workOrder.title,
      confirmCount,
      totalEmployees,
      Math.round(confirmPercentage)
    );
    // ↓ 내부적으로:
    // - 회사 관리자 (BUYER 역할) 조회
    // - 알림 생성: "✅ 업무 '{title}' 완료율: {percentage}%"
  }
});
```

**결과**: 50% 이상의 직원이 업무를 완료하면 관리자에게 알림이 갑니다.

## 3. 장애인직원공지관리 (Announcement) 알림

### 현재 상태: ✅ **완전 구현됨**

공지사항 알림도 업무지시와 동일하게 양방향으로 작동합니다:

### A. 관리자 → 직원 알림

```typescript
// apps/api/src/routes/announcements.ts
router.post('/', requireAuth, async (req, res) => {
  // 1. 공지사항 생성
  const announcement = await prisma.companyAnnouncement.create({
    data: { title, content, companyId, buyerId }
  });
  
  // 2. 모든 직원에게 알림 전송
  await notifyNewAnnouncement(
    announcement.buyerId,
    announcement.id,
    announcement.title
  );
});
```

### B. 직원 → 관리자 알림 (50% 읽음 시)

```typescript
// apps/api/src/routes/announcements.ts
router.post('/:id/read', requireAuth, async (req, res) => {
  // 1. 직원이 공지 읽음 처리
  const readLog = await prisma.announcementReadLog.create({
    data: { announcementId, companyId, buyerId, employeeId, userId }
  });
  
  // 2. 읽음율 계산
  const totalEmployees = await prisma.disabledEmployee.count({
    where: { buyerId: announcement.buyerId }
  });
  
  const readCount = await prisma.announcementReadLog.count({
    where: { announcementId: id }
  });
  
  const readPercentage = (readCount / totalEmployees) * 100;
  
  // 3. 50% 이상 읽음 시 관리자에게 알림
  if (readPercentage >= 50 && !alreadyNotified) {
    await notifyAnnouncementRead(
      announcement.buyerId,
      announcement.id,
      announcement.title,
      readCount,
      totalEmployees,
      Math.round(readPercentage)
    );
  }
});
```

## 요약

### ✅ 완전 작동하는 알림:

1. **휴가 관리**
   - 직원 신청 → 관리자 LEAVE_REQUEST 알림
   - 관리자 승인/거부 → 직원 LEAVE_APPROVED/REJECTED 알림

2. **공지사항 관리**
   - 관리자 생성 → 직원 전체 ANNOUNCEMENT 알림
   - 직원 50% 읽음 → 관리자 "공지 확인 현황" 알림

3. **업무지시 관리**
   - 관리자 생성 → 직원 전체 WORK_ORDER 알림
   - 직원 50% 완료 → 관리자 "업무 완료 현황" 알림

### ⚠️ 구현 필요:

4. **근태 관리**
   - 지각 → 관리자 ATTENDANCE_ISSUE 알림
   - 조퇴 → 관리자 ATTENDANCE_ISSUE 알림
   - 무단결근 → 관리자 ATTENDANCE_REMINDER 알림

## 테스트 방법

### 업무지시 알림 테스트:

```bash
# 1. 관리자 로그인
MANAGER_TOKEN=$(curl -s -X POST http://localhost:4000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"identifier":"01010000001","password":"test1234"}' \
  | grep -o '"accessToken":"[^"]*"' | cut -d'"' -f4)

# 2. 관리자 알림 확인 (Before)
curl "http://localhost:4000/notifications/unread-count?byType=true" \
  -H "Authorization: Bearer $MANAGER_TOKEN"

# 3. 직원 5명이 첫 번째 업무 완료 처리
for phone in 01010010001 01010010002 01010010003 01010010004 01010010005; do
  EMPLOYEE_TOKEN=$(curl -s -X POST http://localhost:4000/auth/login \
    -H "Content-Type: application/json" \
    -d "{\"identifier\":\"$phone\",\"password\":\"test1234\"}" \
    | grep -o '"accessToken":"[^"]*"' | cut -d'"' -f4)
  
  # 첫 번째 업무 ID 조회
  WORK_ORDER_ID=$(curl -s "http://localhost:4000/work-orders?limit=1" \
    -H "Authorization: Bearer $EMPLOYEE_TOKEN" \
    | grep -o '"id":"[^"]*"' | head -1 | cut -d'"' -f4)
  
  # 업무 완료 처리
  curl -s -X POST "http://localhost:4000/work-orders/$WORK_ORDER_ID/confirm" \
    -H "Authorization: Bearer $EMPLOYEE_TOKEN" \
    -H "Content-Type: application/json" \
    -d '{"note":"완료했습니다"}'
done

# 4. 관리자 알림 재확인 (After)
# → WORK_ORDER 타입 알림 1개 증가 확인
curl "http://localhost:4000/notifications/unread-count?byType=true" \
  -H "Authorization: Bearer $MANAGER_TOKEN"
```

**예상 결과**:
- Before: `{ "WORK_ORDER": 0, ... }`
- After: `{ "WORK_ORDER": 1, ... }` (50% 완료 알림)
