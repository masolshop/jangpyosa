# 🚨 배포 대안 방법 (SSH 접속 불가)

## 현재 상황

- ✅ 서버 HTTPS 접근 가능 (https://jangpyosa.com - HTTP 200)
- ❌ SSH 포트 22 접근 불가 (Connection timeout)
- ✅ 코드 수정 완료 및 Git 푸시 완료 (커밋 ec59e24)

## 🔧 배포 대안 방법

### 방법 1: AWS/클라우드 웹 콘솔 사용 (권장) ⭐

서버가 AWS EC2에 호스팅되어 있다면:

1. **AWS 콘솔 접속**
   - https://console.aws.amazon.com/ec2/
   - 서울 리전 선택 (ap-northeast-2)

2. **인스턴스 찾기**
   - IP: 15.164.103.96
   - 또는 Name tag: jangpyosa

3. **Session Manager 사용**
   - 인스턴스 선택 → Connect → Session Manager
   - 또는 EC2 Instance Connect 사용

4. **배포 명령 실행**
   ```bash
   sudo su - ubuntu
   cd /home/ubuntu/jangpyosa
   git pull origin main
   pm2 restart jangpyosa-api
   pm2 logs jangpyosa-api --lines 20
   ```

### 방법 2: 서버 관리자에게 요청

서버 관리자 또는 DevOps 팀에게 다음 스크립트 실행 요청:

```bash
#!/bin/bash
cd /home/ubuntu/jangpyosa
git pull origin main
pm2 restart jangpyosa-api
pm2 status
```

### 방법 3: CI/CD 파이프라인 트리거

GitHub Actions가 설정되어 있다면:

1. Repository → Actions
2. 배포 워크플로우 수동 실행
3. 또는 특정 브랜치로 푸시하여 자동 트리거

### 방법 4: Webhook 사용

서버에 Git webhook이 설정되어 있다면:
- Git 푸시 시 자동 배포될 수 있음
- 확인 방법: Repository → Settings → Webhooks

### 방법 5: 다른 SSH 클라이언트 사용

로컬 컴퓨터에서:

**Windows:**
```bash
# PuTTY 사용
plink -i jangpyosa_deploy.ppk ubuntu@15.164.103.96 "cd /home/ubuntu/jangpyosa && git pull origin main && pm2 restart jangpyosa-api"
```

**Mac/Linux:**
```bash
# 터미널 사용
ssh -i ~/.ssh/jangpyosa_deploy ubuntu@15.164.103.96 "cd /home/ubuntu/jangpyosa && git pull origin main && pm2 restart jangpyosa-api"
```

## 📝 SSH 키 파일 위치
