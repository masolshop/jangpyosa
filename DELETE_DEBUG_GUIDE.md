## 🧪 테스트 방법 (중요!)

### 1단계: 브라우저 강력 새로고침
먼저 **반드시** 캐시를 지우고 페이지를 새로고침하세요:

- **Windows:** `Ctrl + Shift + R`
- **Mac:** `Cmd + Shift + R`
- **또는:** `F12` → Network 탭 → "Disable cache" 체크 → 새로고침

### 2단계: 개발자 도구 열기
1. `F12` 키 누르기 (또는 우클릭 → 검사)
2. **Console** 탭 클릭
3. 콘솔 내용을 **클리어** (🚫 아이콘 클릭)

### 3단계: 삭제 테스트
1. https://jangpyosa.com/admin/sales-management 접속
2. **하위 조직이 없는** 사람 선택 (중요!)
   - 매니저(일반 사원) 선택 권장
   - 본부장/지사장은 하위 조직이 있을 수 있음
3. 🗑️ **삭제** 버튼 클릭
4. 모달에서 **"삭제"** 버튼 클릭 (빨간색 버튼)

### 4단계: 콘솔 확인
**다음과 같은 로그가 나타나야 합니다:**

#### ✅ 성공 케이스:
```javascript
[DELETE] Sending delete request for: <ID> <이름>
[DELETE] Response: { success: true, message: '삭제되었습니다' }
```
→ 화면에 "삭제되었습니다" 메시지 표시
→ 목록에서 해당 인원 사라짐

#### ❌ 실패 케이스 1: 하위 조직 있음
```javascript
[DELETE] Sending delete request for: <ID> <이름>
[DELETE] Error: 하위 조직이 있어 삭제할 수 없습니다...
```
→ 화면에 에러 메시지 표시
→ 목록은 변경 없음

#### ❌ 실패 케이스 2: 인증 문제
```javascript
[DELETE] Sending delete request for: <ID> <이름>
[DELETE] Error: 401 Unauthorized 또는 403 Forbidden
```
→ 다시 로그인 필요

#### ❌ 실패 케이스 3: 네트워크 에러
```javascript
[DELETE] Sending delete request for: <ID> <이름>
[DELETE] Error: Failed to fetch
```
→ 인터넷 연결 확인

## 📸 스크린샷 요청

삭제가 안 되면 다음을 캡처해주세요:

### 1. 브라우저 콘솔 (필수!)
- F12 → Console 탭
- 삭제 버튼 클릭 후 나타나는 **모든 로그**
- `[DELETE]` 로 시작하는 메시지 포함

### 2. 네트워크 탭 (필수!)
- F12 → Network 탭
- DELETE 요청 찾기 (people/<ID>)
- 클릭 후:
  - **Headers 탭:** Request URL, Status Code 확인
  - **Response 탭:** 응답 내용 확인
  - **Preview 탭:** JSON 데이터 확인

### 3. 화면 에러 메시지
- 삭제 시도 후 화면 상단에 표시되는 에러 메시지

## 🔍 예상 원인별 해결 방법

### 원인 1: 하위 조직이 있는 경우 ⚠️
**증상:** "하위 조직이 있어 삭제할 수 없습니다" 에러

**해결:**
1. **매니저(일반 사원)** 먼저 삭제해보세요
2. 또는 하위 직원을 다른 곳으로 **이동**(🔄 버튼)
3. 그 후 상위 관리자 삭제

**확인 방법:**
- 조직도 보기에서 해당 인원 확인
- 하위 인원이 보이면 먼저 처리

### 원인 2: 토큰 만료 🔐
**증상:** 403 또는 401 에러

**해결:**
1. 로그아웃
2. 다시 로그인 (ID: 01063529091 / PW: admin123)
3. 영업 관리 페이지 다시 접속
4. 삭제 재시도

### 원인 3: 요청이 전송되지 않음 🚫
**증상:** 콘솔에 `[DELETE] Sending...` 로그가 없음

**원인:**
- 모달 버튼이 작동하지 않음
- JavaScript 에러

**해결:**
1. 페이지 강력 새로고침 (Ctrl+Shift+R)
2. 콘솔에서 JavaScript 에러 확인
3. 에러 스크린샷 공유

### 원인 4: DELETE 요청은 가지만 실패 ❌
**증상:** 
- 콘솔: `[DELETE] Sending...` ✅
- 네트워크: DELETE 요청 존재 ✅
- 하지만 Status Code가 400/500

**확인:**
1. Network 탭에서 Status Code 확인
2. Response 내용 확인
3. 에러 메시지 스크린샷

## 💡 빠른 테스트 팁

### 삭제 가능한 대상 찾기:
1. [목록 보기] 탭
2. **역할**이 "매니저"인 사람 찾기
3. 해당 매니저 삭제 시도

### 왜 매니저인가?
- 본부장 → 지사장들이 소속
- 지사장 → 매니저들이 소속
- **매니저** → 하위 조직 없음 (삭제 가능!)
