## 🔍 테스트 시나리오

### Step 1: 로그인
1. https://jangpyosa.com 접속
2. **페마연구소** 관리자 계정으로 로그인
   - 이메일: `admin@pema.com`
   - (비밀번호는 기존 사용하던 것)

### Step 2: 업무지시 목록 확인
1. 좌측 메뉴에서 **"📋 업무지시"** 또는 **"장애인직원업무관리"** 클릭
2. **"테스트 업무지시 - 월말 점검"** 항목 확인
   - ✅ 제목이 표시되어야 함
   - ✅ 수신자 3명 표시
   - ✅ 완료 1명 (33%) 표시

### Step 3: 상세보기 클릭 (핵심 테스트!)
1. **"상세보기"** 버튼 클릭
2. ✅ **확인사항**:
   - 모달/팝업이 열려야 함
   - 제목: "테스트 업무지시 - 월말 점검" 표시
   - 내용: "월말 재고 점검 및 정리..." 전체 내용 표시
   - 작성자: "김관리자" 표시

## 🐛 만약 오류 발생 시

### 404 오류가 발생하면:
1. 브라우저 콘솔(F12)에서 실제 요청 URL 확인
2. 스크린샷 캡처 (전체 화면 + 콘솔)
3. 다음 명령으로 로그 확인:
```bash
pm2 logs jangpyosa-api --lines 50
```

### 수신자 이름이 "알 수 없음"으로 표시되면:
- User 테이블과 WorkOrderRecipient의 userId 연결 문제
- API 로그에서 "Successfully retrieved work order" 메시지 확인

### 모달이 열리지 않으면:
- 프론트엔드 `loadWorkOrderDetail` 함수 오류
- 브라우저 콘솔에 에러 메시지 확인
