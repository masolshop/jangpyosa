### 3단계: 근본 원인 발견
**문제**: `getUnreadCountByTypes` 함수가 새로운 알림 타입을 인식하지 못함

실제 알림 타입:
- ✅ `WORK_ORDER` - 새 업무 생성 시
- ❌ `WORK_ORDER_COMPLETED` - 업무 완료 시 (누락!)
- ✅ `ANNOUNCEMENT` - 새 공지 생성 시
- ❌ `ANNOUNCEMENT_READ` - 공지 확인 시 (누락!)

## 🛠️ 해결 방법

### 수정 코드
**파일**: `apps/api/src/services/notificationService.ts`

```typescript
export async function getUnreadCountByTypes(userId: string) {
  // ... groupBy 로직 ...
  
  return {
    total,
    byType: {
      LEAVE_REQUEST: byType.LEAVE_REQUEST || 0,
      LEAVE_APPROVED: byType.LEAVE_APPROVED || 0,
      LEAVE_REJECTED: byType.LEAVE_REJECTED || 0,
      WORK_ORDER: byType.WORK_ORDER || 0,
      WORK_ORDER_COMPLETED: byType.WORK_ORDER_COMPLETED || 0,  // ✅ 추가
      ANNOUNCEMENT: byType.ANNOUNCEMENT || 0,
      ANNOUNCEMENT_READ: byType.ANNOUNCEMENT_READ || 0,  // ✅ 추가
      ATTENDANCE_REMINDER: byType.ATTENDANCE_REMINDER || 0,
      ATTENDANCE_ISSUE: byType.ATTENDANCE_ISSUE || 0,
    },
  };
}
```

| 알림 유형 | 트리거 | 수신자 | 타입 | 상태 |
|----------|--------|--------|------|------|
| 휴가 신청 | 직원이 신청 | 관리자 | `LEAVE_REQUEST` | ✅ |
| 휴가 승인 | 관리자가 승인 | 직원 | `LEAVE_APPROVED` | ✅ |
| 휴가 거부 | 관리자가 거부 | 직원 | `LEAVE_REJECTED` | ✅ |
| 공지 생성 | 관리자가 생성 | 전체 직원 | `ANNOUNCEMENT` | ✅ |
| 공지 50% 읽음 | 50% 이상 읽음 | 관리자 | `ANNOUNCEMENT_READ` | ✅ 수정됨 |
| 업무 생성 | 관리자가 생성 | 전체 직원 | `WORK_ORDER` | ✅ |
| 업무 50% 완료 | 50% 이상 완료 | 관리자 | `WORK_ORDER_COMPLETED` | ✅ 수정됨 |

## 🎯 결론

**문제**: 관리자에게 알림이 안 뜨는 것으로 보고됨

**실제 상황**:
1. ✅ 알림은 DB에 정상 생성됨
2. ✅ API는 정상 작동함
3. ❌ `getUnreadCountByTypes` 함수가 새 타입을 인식하지 못함

**해결**:
1. ✅ `WORK_ORDER_COMPLETED` 타입 추가
2. ✅ `ANNOUNCEMENT_READ` 타입 추가
3. ✅ 프로덕션 배포 완료

**결과**: 관리자가 이제 정확히 알림을 볼 수 있습니다! 🎉
